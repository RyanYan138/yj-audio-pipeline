# YJ Audio Pipeline 使用流程（2026-05-26）

> 适用环境：华为云 Notebook / 任意有 GPU 的 Linux 服务器  
> 前提：已安装 conda，有 CUDA 驱动

---

## 一、第一次使用（只需做一次）

### 1.1 克隆代码

```bash
git clone git@github.com:RyanYan138/yj-audio-pipeline.git
cd yj-audio-pipeline
```

### 1.2 配置 conda 环境

激活你已有的环境（华为服务器用 `vllm-qwen`，其他服务器自行替换）：

```bash
conda activate vllm-qwen
```

运行一键安装脚本（补装 pipeline 缺少的包）：

```bash
bash setup_hwserver.sh
```

脚本会自动安装：`faster-whisper`、`ctranslate2`、`silero-vad`、`modelscope`、`funasr`、`onnxruntime-gpu` 等。

### 1.3 下载模型

在项目根目录下创建 `models/` 目录并下载模型：

```bash
mkdir -p models/faster-whisper-large-v3
mkdir -p models/FunASR-Nano
# 如果要用 Whisper HuggingFace 版（run_all4user_nodocker.sh 需要）
mkdir -p models/whisper-large-v3
```

**下载 faster-whisper-large-v3**（LID 语言识别 + Whisper 转写都用这个，必须下载）：

```bash
huggingface-cli download Systran/faster-whisper-large-v3 \
    --local-dir models/faster-whisper-large-v3
```

**下载 FunASR Nano**（如果要用 FunASR 版本才需要）：

```bash
modelscope download --model FunAudioLLM/Fun-ASR-Nano-2512 \
    --local_dir models/FunASR-Nano
```

**下载 Whisper HuggingFace 版**（如果要用 `run_all4user_nodocker.sh` 才需要）：

```bash
modelscope download --model AI-ModelScope/whisper-large-v3 \
    --local_dir models/whisper-large-v3
```

下载完后目录结构应该是：

```
yj-audio-pipeline/
├── models/
│   ├── faster-whisper-large-v3/   ← 必须有
│   ├── FunASR-Nano/               ← 用 FunASR 时需要
│   └── whisper-large-v3/          ← 用原版 Whisper 时需要
├── FireRedVAD/                    ← 已内置，不需要下载
├── test/audio/                    ← 自带 4 个测试音频
...
```

## 二、修改配置（每次跑之前）

每个脚本顶部都有一个「用户主要配置区」，**只需要改这里**，其他不要动。

以 `run_silero_whisper_fast.sh` 为例：

```bash
# ========== 修改这里 ==========

# 输入音频目录（你的原始音频放在哪里）
INPUT_ROOT="/your/raw/audio/path"

# 输出目录（结果保存到哪里）
OUTPUT_ROOT="/your/output/path"

# 使用哪几张 GPU（填卡号，从 0 开始）
GPUS="0,1,2,3"

# 语言过滤（只保留哪些语言）
LANG_TARGET_LANGS=("zh" "en")

# ==============================
```

> **提示**：用 `nvidia-smi` 查看有哪些 GPU 可用，填对应编号。

## 三、选择脚本

根据需求选择对应脚本：

| 脚本名 | VAD | ASR | 速度 | 推荐场景 |
|---|---|---|---|---|
| `run_silero_whisper_fast.sh` | Silero | faster-whisper | ⭐⭐⭐ 最快 | **推荐首选，提速65%** |
| `run_all4user_nodocker.sh` | Silero | Whisper HF 版 | ⭐⭐ | 稳定，兼容性好 |
| `run_fireredvad_whisper.sh` | FireRedVAD | faster-whisper | ⭐⭐ | VAD 精度更高 |
| `run_fireredvad_funasr.sh` | FireRedVAD | FunASR Nano | ⭐⭐ | 全国产模型 |
| `run_fireredvad_funasr_fast.sh` | FireRedVAD | FunASR Nano | ⭐ | 流水线版（FunASR 逐条推理，提速不明显）|

**模型需求对照：**

| 脚本 | 需要的模型 |
|---|---|
| `run_silero_whisper_fast.sh` | `models/faster-whisper-large-v3` |
| `run_all4user_nodocker.sh` | `models/whisper-large-v3` + `models/faster-whisper-large-v3` |
| `run_fireredvad_whisper.sh` | `models/faster-whisper-large-v3` |
| `run_fireredvad_funasr.sh` | `models/FunASR-Nano` + `models/faster-whisper-large-v3` |
| `run_fireredvad_funasr_fast.sh` | `models/FunASR-Nano` + `models/faster-whisper-large-v3` |

## 六、常见问题

**Q：运行报 `ModuleNotFoundError`**  
A：重新运行 `bash setup_hwserver.sh` 补装缺失包。

**Q：报 `No such file or directory: models/xxx`**  
A：对应模型没下载，按第一步的说明下载到对应目录。

**Q：GPU 报 `CUDA out of memory`**  
A：减少 `GPUS` 里的卡数，或调小 `WHISPER_BATCH_SIZE`（默认8，可改为4）。

**Q：export=0，没有输出**  
A：通常是 DNSMOS 阈值过滤掉了所有音频。检查音频质量，或临时降低阈值：
```bash
DNSMOS_MIN_MOS_OVR="1.5"   # 默认2.0，可以降低
DNSMOS_MIN_MOS_BAK="2.0"   # 默认3.5，可以降低
```

**Q：速度很慢**  
A：优先使用 `run_silero_whisper_fast.sh`（流水线版），确保 `GPUS` 配了多张卡。

## 七、快速测试（验证环境）

用项目自带的 4 个测试音频跑一遍，确认环境没问题：

```bash
# 脚本默认 INPUT_ROOT 就指向 test/audio/，直接跑即可
bash run_silero_whisper_fast.sh
```

跑完后检查：

```bash
cat output/test_run_silero_whisper_fast/final_dataset/labels.json
```

如果能看到中英文转写结果，说明环境 OK。
